Tensile Deformation


Overview

In the previous lecture, we learned about MD for modeling shear deformation in materials. In materials science, tensile deformation, referring to deforming the material by stretching it, is equally important to shear deformation.

In experimental materials science research, tensile deformation is a frequently used method for analyzing a material's ductility (ease of elongation) and fracture strength. Fracture is a common phenomenon in real life, occurring in social infrastructure such as bridges and roads due to the effects of aging, earthquakes and other factors.

While fracture is observed as a macroscopic phenomenon visible to the naked eye, its physical essence is ascribed to the dissociation of interatomic bonds. Direct experimental observation of this phenomenon's fundamental nature remains beyond current scientific capabilities. Consequently, numerous fundamental questions about fracture remain unanswered: Why did something that had never broken before suddenly break at a specific moment?; Why did fracture occur at that specific location rather than elsewhere? Therefore, research into the mechanisms of fracture is a field where the application of MD to model atomic-level behavior is strongly desired.

Running MD for Tensile Deformation

When performing MD simulations of tensile deformation, it is typical to introduce notches or other initiation points for fracture beforehand. However, as we are focusing solely on the method of performing this deformation via MD, as in the previous case, we will study a simple example of deforming a perfect crystalline iron sample.

The script for this MD is bccFe_tensile.lcm in script directory. You can execute it using the following command. 
$work> lmp_serial -in script/bccFe_tensile.lcm
This will output numerous files in the format bccFe_tensile_*.out. When you load one of these files, for example, the first file bccFe_tensile_0.out, into OVITO, it will automatically load all the other files in the same format. Click the play button at the bottom of the OVITO screen to run the animation. If you can observe the material stretching vertically as shown in the video below, the execution should be successful.



Step-by-Step Analysis of Input Script

Create the Target Crystal

Here is the first block of the script: 
units metal
boundary p p p
atom_style atomic
variable 	lc equal 2.83
lattice bcc ${lc} orient x 1 0 0 orient y 0 1 0 orient z 0 0 1
variable 	x_box equal 30*${lc}
variable 	y_box equal 30*${lc}
variable 	z_box equal 50*${lc}
region box block 0 ${x_box} 0 ${y_box} 0 ${z_box} units box
create_box 1 box
create_atoms 1 box
We've studied units, boundary, and atom_style commands. Audience who is unfamiliar with these should return to the lecture on Structure Relaxation to review them.

The following variable command defines a variable named lc. As we'll see later, this is the lattice constant for the iron crystal.

The following lattice command is the command for defining the lattice system. We encountered this command in a few previous lectures

The following three variable commands define the variables x_box, y_box, and z_box as 30 times the lattice constant.

The following region command has been mentioned before, and we define a rectangular region named box. Note that the last argument is units box. Previously, units lattice was selected instead, in which case the length unit was the lattice constant. When units box is specified, the length unit used by this command becomes Angstrom.

When defining the simulation box, it is possible to use either the lattice constant as in previous lectures or Angstrom as in this lecture. There is no fundamental difference between the two methods. Generally, as scripts become more complex, using Angstrom tends to be easier to develop the script.

The subsequent two commands have already appeared in previous lectures. The create_box command creates a simulation box from the region called box created above, and the create_atoms command places type-1 atoms within that box to form the crystal structure specified by the lattice command above.

Defining Atomic Interaction

The next block of script is as follows: 
pair_style eam/fs
pair_coeff * * potentials/Fe_mm.eam.fs Fe
neigh_modify every 1 delay 0 check yes
Here, we define the interatomic potential and updating the list of the neighbor list used to calculate interatomic forces. As we've already saw this block in previous lectures, we'll move on without detailed explanation.

Structural Relaxation

The next block is : 
variable 	dump_step equal 200
thermo ${dump_step}
thermo_style custom step pe lx ly lz press
min_style cg
min_modify dmax 0.05
minimize 1.0e-25. 1.0e-12 10000 50000
In this block, we are performing structural relaxation, as mentioned several times before. First, we defined a variable named dump_step using variable command. The thermo command specifies the frequency at which thermodynamic variables are calculated and output during the MD simulation. The subsequent thermo_style command specifies which variables to output to standard output. Here, these are the time step, total potential energy, the box size in the x, y, and z directions, and pressure.

The following min_style command specifies the structure relaxation algorithm; here, the conjugate gradient method is used.

The min_modify dmax command specifies the upper limit for the speed of structural relaxation. Here, it is assumed that each atomic position will not change by more than 0.05 Angstroms per iteration. If structural relaxation is not proceeding well, this value should be smaller; if its convergence is too slow, a larger value may be used. Since the default value is 0.1 Angstrom, this command should only be used when you want to specify a different value as in this script.

The minimize command at the bottom executes structural relaxation. The four arguments were explained in the lecture for Structural Relaxation, so refer to the previous lecture if necessary.

Variable used for tensile deformation

The previous block gives us the initial state for performing the tensile deformation. Now define the necessary variables for the deformation procedure. 
variable 	lx_current equal lx
variable 	ly_current equal ly
variable 	lz_current equal lz
variable 	lx0 equal ${lx_current}
variable 	ly0 equal ${ly_current}
variable 	lz0 equal ${lz_current}
variable 	dstrain equal 0.001
variable	poisson_ratio equal 0.27
variable 	lz_delta equal ${dstrain}*${lz0}/2
variable 	lx_delta equal ${lz_delta}*${poisson_ratio}
variable 	ly_delta equal ${lz_delta}*${poisson_ratio}
lx_current, ly_current, and lz_current represent the box size, i.e., the material dimensions. lx0, ly0, and lz0 are defined as the same values. As will be explained later, lx_current and the others denote the current values during tensile deformation, while lx0 and the others represent the constant values, i.e., the dimensions measured immediately after structural relaxation.

In this example, we do not use methods used in the previous lecture for shear deformation, where we applied a forced velocity to a portion of the sample after annealing it. Here, we adopt a method where the material is deformed at infinite speed (i.e. jumping to the next shape) in tiny increments at 0 K, and allowing structural relaxation after the tiny deformation. The process is repeated till the final deformation reaches the requested value.

The variable dstrain defined in the next line represents the strain rate per step, meaning it will pull and deform the initial z-direction length lz0 by 0.1% at a time.

The following poisson_ratio is the Poisson's ratio of the material. When an iron crystal is stretched in the z-direction, it contracts in the transverse direction; the Poisson's ratio indicates this ratio, and the value in the literature is given here.

The incremental deformation is achieved by moving the box's boundaries in three directions by a small amount. lz_delta, lx_delta, and ly_delta represent the amount moved per increment. The z-direction value is divided by 2 because both the top and bottom boundaries are moved, resulting in the vertical length increasing by lz_delta * 2 per increment. Additionally, the Poisson ratio is used in the definitions of lx_delta and ly_delta.

Running Tensile Deformation

Here we show the rest of the script: 
reset_timestep 0
dump 1 all custom ${dump_step} bccFe_tensile_*.out id type xs ys zs
variable final_strain equal 0.04
variable nloop equal ${final_strain}/${dstrain}
variable loop loop ${nloop}
label loop_start
	print " "
	print " "
	print "%%%%% LOOP INCREMENT ${loop} %%%%"
	change_box all x delta ${lx_delta} -${lx_delta} y delta ${ly_delta} -${ly_delta} &
	z delta -${lz_delta} ${lz_delta} remap units box
	minimize 1.0e-25 1.0e-25 1000 10000
	run 0
	next loop
jump SELF loop_start
reset_timestep command at the top, which has appeared several times before, initializes the time.

dump ... custom command has appeared several times so far; it is a command for outputting the intermediate states of each atom to a file.

The variable final_strain defines the final elongation achieved by the whole tensile simulation. Here, we set the final elongation to 4%.

The variable nloop represents the total number of the tiny deformation.

The following variable ... loop command is new. It defines a variable incremented during iterative calculations using the syntax variable ??? loop $$$. Here, ??? is the variable name, whose initial value is set to 1. $$$ represents the final value of the loop variable during the iteration process.

The label command specifies the destination of the jump command at the bottom. The explanation given below will clarify how this label command works in iterative calculations, so let's proceed.

The indented section that follows is the repeating part. The first three print commands display the intermediate results of this iterative calculation to standard output.

The following two lines are connected by "&" and form a single sentence. The change_box command forcibly moves the box boundaries. When the keywords x delta, y delta, and z delta are specified, they define the movement distances for the box's lower and upper boundaries, respectively. Here, we see that in the x and y directions, the lower boundary increases and the upper boundary decreases, causing compression, while in the z direction, the opposite occurs, causing extension. The remap keyword in this command requires atomic positions to be changed proportionally to the degree of deformation. The subsequent units box indicates that the length unit used in this command is Angstrom. This is the implementation of the tiny tensile deformation mentioned above.

minimize is a command that relaxes the structure after applying the deformation.

The next run 0" is a bit tricky way to use run. So far the run commands were used to perform dynamic calculations for a finite number of steps, but the argument is 0, which seems odd. In this case, the run command is used to calculate the thermodynamic variables of the system without advancing time

By next loop command, the variable loop is incremented.

jump command at the bottom is used to jump to a specific location of a script. If SELF is specified as the first argument, the jump occurs within the same script. If another script is specified as the first argument, it moves to that script. The second argument specifies the destination label of the jump. In this example, it returns to the beginning of this iterative calculation loop. However, if the previous next command caused "loop" to exceed its bounds (here, nloop), this statement is ignored and the iterative calculation terminates. In this case, since it is the end of the entire script, LAMMPS execution ends.

Note for deformation method in MDs

In the previous lecture, we introduced the script that performs shear deformation. That method involved forcibly applying velocity to a portion of the material to induce deformation. It is also possible to apply that method to the tensile deformation.

The method of applying velocity inevitably leads to the technical problem that the deformation rate becomes unrealistically fast compared to the deformation applied in experiments. Deformation time in experiments typically lasts at least seconds. However, the time that can be tracked in MD is at most on the order of nanoseconds. One solution is to repeatedly perform the procedure of applying a tiny deformation at 0 K and allowing it to relax, as done in this study. In this case, since the structure is allowed to relax after each deformation, it avoids criticism that the deformation rate is too fast and unrealistic.

However, those methods cannot be used at finite temperatures, so they cannot be applied in cases where lattice vibrations largely affect material behavior. In reality, materials become more prone to plastic deformation as temperature increases, and fracture is suppressed. Therefore, MD simulations that deform extremely slowly at finite temperatures would be ideal, but even the latest supercomputers have limitations in achieving this. In other words, whichever method is chosen, there are pros and cons.

Finite-temperature deformation MD may reflect some reality, but blindly believing the MD results can lead to incorrect conclusion. Thus you need careful analyses such as comparison with experiments.
